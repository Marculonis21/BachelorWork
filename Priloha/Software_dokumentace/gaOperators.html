gaOperators module

Implemented genetic operators

This module contains a single class Operators in which there are many already fully implemented genetic operators ready to be used in evolutionary algorithms.

Those operators are implemented as static methods so user has to create only an instance of this class to be able to use operators.

Default implemented operators:
Selection:

  • roulette_selection()

  • tournament_selection()

  • tournament_prob_selection()

Crossover:

  • crossover_single_point()

  • crossover_uniform()

Mutation:

  • uniform_mutation()

  • uniform_shift_mutation()

Note

If one wants to implement custom operators, it is adviced to do so inside this Operators class and name the function in a way that the name includes one of the types of operators (i.e. ‘selection’, ‘crossover’ or ‘mutation’).

There is a methods Operators._ops_dir() which is used to automatically detect all operator methods and sort them by type to be later used inside GUI without any additional work necessary.

class gaOperators.Operators
static _ops_dir()

Method for automatic collection of implemented genetic operators from within this class.

Returns:

dictionary of all recognised genetic operators by operator type and operator name.

Return type:

dict[str, dict[str, (func, func params*)]]

Warning

For methods to be correctly recognize by this function, their names must include the full name of the operator type which they are implementing (i.e. ‘selection’, ‘crossover’ or ‘mutation’).

static roulette_selection(population, fitness_values)

Runs basic roulette selection between all individuals with probability of selection corresponding to their fitness (needs all positive fitness values).

Returns:

List of selected parents of the same size as original population.

Return type:

List[individual]

static tournament_selection(population, fitness_values, k=5)

Runs tournamnets between randomly chosen individuals and selects the best from each tournament.

Returns:

List of selected parents of the same size as original population.

Return type:

List[individual]

static tournament_prob_selection(population, fitness_values, probability=0.9, k=5)

Runs tournamnets between randomly chosen individuals and selects one according to probability based on their results. For each individual with tournament placement X:

\[p(X) = probability*(1-probability)^{X-1}\]

(These probabilities are then normalised so they sum up to 1)

Returns:

List of selected parents of the same size as original population.

Return type:

List[individual]

static crossover_single_point(population, agent)

Single point crossover operator. Uses agent attributes - BaseAgent.evolve_control and BaseAgent.evolve_body.

Returns:

List of created offsprings of the same size as original population.

Return type:

List[individual]

static crossover_uniform(population, agent)

Uniform crossover operator - selects genetic information from each parent with probability 1/2. Uses agent attributes - BaseAgent.evolve_control and BaseAgent.evolve_body.

Returns:

List of created offsprings of the same size as original population.

Return type:

List[individual]

static uniform_mutation(population, agent)

Uniform mutation operator - with set probabilities (set by agent attributes - BaseAgent.individual_mutation_prob, BaseAgent.action_mutation_prob and BaseAgent.body_mutation_prob) mutates a value to to new from allowed range (allowed ranges of values are also set by agent attributes - BaseAgent.action_range and BaseAgent.body_range).

Returns:

List of mutatted offsprings.

Return type:

List[individual]

static uniform_shift_mutation(population, agent, max_shift_percentage=0.05)

Uniform shift mutation operator - works similarly to uniform_mutation() but instead of generating random value from the whole allowed range when mutation value, it only generates value in small part of allowed range (argument max_shift_percentage) and changes current value by this small generated shift.

Returns:

List of mutatted offsprings.

Return type:

List[individual]
