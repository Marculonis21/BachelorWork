gaAgents module

Module for genetic agents.

Module that stores base agent abstract class BaseAgent from which all the other agent classes have to inherit.

Agent class is heavily used as an information source for GUI.

Usage example:

import gaAgents

robot = ... # Get some robot (type BaseRobot)

# regular TFSAgent instance
agent1 = gaAgents.TFSAgent(robot)

# TFSAgent with unlocked robot body parts (first with range 1 to 5, last with range 2 to 3)
agent2 = gaAgents.TFSAgent(robot, [(1,5), False, False, (2,3)])
Default implemented agents:

  • FullRandomAgent

  • StepCycleHalfAgent

  • SineFuncFullAgent

  • SineFuncHalfAgent

  • TFSAgent

  • NEATAgent

enum gaAgents.EvoType(value)

EvoType enum Enum used for naming different types of evolution that agent can perform.

Valid values are as follows:

CONTROL = <EvoType.CONTROL: 0>
BODY = <EvoType.BODY: 1>
CONTROL_BODY_PARALEL = <EvoType.CONTROL_BODY_PARALEL: 2>
CONTROL_BODY_SERIAL = <EvoType.CONTROL_BODY_SERIAL: 3>
class gaAgents.BaseAgent(robot, body_part_mask, evo_type=EvoType.CONTROL, gui=False)

Base agent class

This is the base abstract class from which every agent is inheriting and also calling in their init function.

In __init__() we get to initialise all default class fields which might be later used by the evolutionary algorithm, by the UI or by the agent itself.

Variables:

  • evo_type (EvoType) – Value of selected evolution type.

  • continue_evo (bool) – Flag showing if the evolution of agent will continue after generations are finished (used for serial control-body evolution EvoType.CONTROL_BODY_SERIAL).

  • evolve_control (bool) – Flag showing if robot controls (actions) should be evolved.

  • evolve_body (bool) – Flag showing if robot body should be evolved.

  • body_part_mask (List[False|Tuple[float, float]]) – List of bools and tuples, showing which body parts should be unlocked for evolution and what ranges are allowed for them.

  • orig_body_part_mask – Copy original body_part_mask parameter. Used for evolution type switching.

  • action_range (List[Tuple[float, float]]) – List of tuples (ranges of values) specifying allowed range of values for each action - creator of new agent class is *obligated* to specify this range, used in genetic operators. For examples look into implemented classes.

  • body_range (List[Tuple[float]]) – Same as action_range, but for body parts.

  • action_size – Size of the action space that agents environment supports.

  • observation_size – Size of the observation space that agents environment returns (we often don’t care about this one).

  • gui (bool) – Flag showing if this agent was created through GUI (possibly changing which genetic operators are used).

  • arguments (Dict[str, float|{"MIN":float, "MAX":float}]) – Dictionary of agent specific arguments. Every agent populates this dictionary by itself.

  • arguments_tooltips (Dict[str, str]) – Dictionary of optional tooltips (hints) for arguments values. Used from GUI when describing agent options.

  • genetic_operators (Dict[str, List[Callable, List[Callable parameters ...]]]) – Dictionary of selected genetic operations for selection, crossover and mutation, which overrides default operators when the agent is created via GUI (gui).

  • individual_mutation_prob (float) – Mutation probability - allow mutation of selected individual.

  • action_mutation_prob (float) – Mutation probability - allow mutation of selected action.

  • body_mutation_prob (float) – Mutation probability - allow mutation of selected body part.
